이 프로젝트는 Claude Code, Codex CLI, Gemini CLI의 기능을 확장하기 위한 커스터마이징 도구 모음입니다.

왜 이 프로젝트를 만들었나?

🚀 빠른 시작

5분 안에 시작하기 - 아래 단계를 따라 기본 설정을 완료하세요.

방법 1: 자동 설치 (권장)

저장소 클론

git clone https://github.com/Dannykkh/claude-code-agent-customizations
cd claude-code-agent-customizations

설치 스크립트 실행

# Windows - 인터랙티브 메뉴 (LLM 선택)
install.bat

# Windows - 전체 자동 설치 (모든 LLM + 모든 번들)
install.bat --all

# Linux/Mac
chmod +x install.sh && ./install.sh        # 인터랙티브
./install.sh --all                          # 전체 자동

설치 확인

# Claude Code에서 실행
/skills      # 설치된 스킬 확인
/mcp         # MCP 상태 확인

설치 옵션

옵션 설명 예시
(옵션 없음) 인터랙티브 메뉴 (Up/Down, Space, Enter) install.bat
--all 전체 자동 설치 (모든 LLM + 모든 번들) install.bat --all
--llm <list> 대상 CLI 지정 (콤마 구분) install.bat --llm claude,codex
--uninstall 설정 제거 (MCP, Mnemo, Hooks 등) install.bat --uninstall

설치 모드 비교 (복사 vs 링크)

모드 명령어 장점 단점
복사 (기본) install.bat 원본과 독립적, 안전 업데이트 시 재설치 필요
링크 install-link.bat
./install.sh --link		 git pull만으로 자동 반영 원본 삭제 시 동작 안 함
링크 해제 install-unlink.bat
./install.sh --unlink		 링크 제거 후 원래 상태 복원 -

⚙️ 핵심 시스템

이 프로젝트를 차별화하는 6가지 핵심 시스템입니다.

🎯 Zephermine (젭마인) Planning

24단계 심층 인터뷰로 완전한 SPEC 문서를 생성합니다. 5 Whys 기법, 5-에이전트 리뷰, 자동 검증 포함.

/zephermine → Interview → SPEC.md → 5-Agent 검증

🧠 Mnemo (므네모) Memory

파일 기반 크로스 세션 메모리 시스템. DB 불필요, 훅에서 AI 호출 없음. Claude/Codex/Gemini 3-CLI 통합 기억.

Session A → #tags 저장 → MEMORY.md → Session B 복원

🤖 workpm Multi-AI

PM이 작업 분배, Worker(Claude+Codex+Gemini)가 파일 락킹으로 병렬 실행. Agent Teams 또는 MCP 모드 선택.

workpm (PM) → Task 분배 → pmworker 병렬
CLIPM 명령어특징
ClaudeworkpmAgent Teams, 실시간 팀원 통신
Codex/Geminiworkpm-mcpMCP 도구만, 태스크 기반
공통pmworkerWorker: 태스크 수행, 파일 락

🧪 QPassenger (큐패신저) QA

QA 시나리오를 Playwright 테스트로 변환, 모든 테스트 통과까지 자동 수정 반복 (Healer Loop, max 5회).

시나리오 수집 → Playwright 생성 → 실행 → Healer Loop

🔄 Chronos (크로노스) Auto Loop

Chronos(시간의 신) — 끝없이 돌아가는 수레바퀴. 서브에이전트가 이슈 찾기→수정→검증을 자율 반복. 사용자 개입 0회.

FIND → FIX → VERIFY → 반복 (자율)

⚡ Zeus (제우스) Full Pipeline

한 줄 설명만으로 설계→구현→테스트 전체를 자동 완료. AskUserQuestion 절대 호출 금지. 5개 Phase 연속 실행.

/zeus → Zephermine → workpm → QPassenger → Report

추천 워크플로우

시나리오 체이닝 순서
새 프로젝트 (풀코스)zephermine → architect → agent-team → qpassenger → docker-deploy
전자동 (제우스)zeus — 한 줄 설명만으로 설계→구현→테스트 완전 자동
기능 추가zephermine → agent-team/수동 구현 → qpassenger
UI 디자인 → 구현stitch-enhance-prompt → stitch-loop → stitch-react → frontend-react
코드 리뷰 종합code-reviewer → security-reviewer → reducing-entropy
QA 자동화qpassenger (시나리오 자동 생성 → Playwright → Healer)

3-Layer 품질 보장 아키텍처

Layer 1: AGENTS.md (Passive Guidelines) → 핵심 규칙이 항상 컨텍스트에 존재 → AI가 처음부터 좋은 코드 작성 Layer 2: Hooks (Automatic Enforcement) → 규칙 위반 자동 감지 (validate-code, validate-api, validate-docs) → 즉시 피드백, 수정 유도 Layer 3: Skills (On-demand Analysis) → 사용자 요청 시 상세 분석 → /code-reviewer, /naming-analyzer, /security-reviewer 등

🔌 MCP 서버

외부 서비스와 연동하기 위한 Model Context Protocol 서버입니다.

주요 MCP 서버

MCP 설명 설치
토스페이먼츠 결제 연동 10분 완료 (PG 업계 최초) claude mcp add tosspayments -- npx -y @tosspayments/integration-guide-mcp@latest
Context7 최신 라이브러리 문서 검색 (Next.js 15, React 19 등) claude mcp add context7 -- npx -y @upstash/context7-mcp
Playwright 브라우저 자동화 claude mcp add playwright -- npx -y @playwright/mcp@latest
Stitch Google Stitch UI 디자인 npx -p stitch-mcp-auto stitch-mcp-auto-setup
GitHub GitHub API 접근 claude mcp add github -- npx -y @modelcontextprotocol/server-github
