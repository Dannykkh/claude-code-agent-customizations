# 프로젝트 명세서 샘플

이 문서는 `markdown_to_pdf.py`의 동작을 검증하기 위한 샘플입니다.
한국어 + 영어 혼용, 코드 블록, 표, 인용 등을 포함합니다.

## 1. 개요

A4 페이지에 25mm 마진을 적용하고, **Pretendard** 본문 폰트와 *D2Coding* 코드 폰트를 사용합니다.
한글이 깨지지 않고 자연스럽게 렌더링되어야 합니다.

> "도구는 인간을 위해 존재하지, 인간이 도구를 위해 존재하지 않는다."
> — 어떤 엔지니어

## 2. 기능 요구사항

### 2.1 기본 변환

다음 마크다운 요소를 PDF로 변환합니다:

- 본문 단락 (한글/영문 혼용)
- 헤딩 (h1 ~ h4)
- 굵게, *기울임*, `인라인 코드`
- 순서 있는/없는 리스트
- 표
- 코드 블록 (신택스 하이라이트)
- 인용
- 수평선

### 2.2 페이지 레이아웃

| 항목 | 값 | 비고 |
|------|------|------|
| 페이지 크기 | A4 | 210×297mm |
| 마진 | 25mm | 한국 출판 표준 |
| 본문 폰트 | Pretendard | 자동 다운로드 |
| 코드 폰트 | D2Coding | 자동 다운로드 |
| 페이지 번호 | 하단 중앙 | "N / M" 형식 |

### 2.3 코드 예시

Python 함수 정의 예시:

```python
def korean_date(d=None):
    """오늘 날짜를 한국식으로 반환합니다."""
    from datetime import date
    d = d or date.today()
    return f"{d.year}년 {d.month}월 {d.day}일"


# 사용 예
print(korean_date())  # 2026년 5월 3일
```

쉘 명령어 예시:

```bash
python markdown_to_pdf.py generate sample.md
# → /path/to/sample.pdf
```

## 3. 트레이드오프

### 3.1 weasyprint vs Paged.js

- **weasyprint (선택)**: Python 네이티브, 한글 잘 됨, GTK 의존
- **Paged.js (gstack)**: Node 기반, 더 풍부한 CSS 지원, 빌드 복잡

### 3.2 폰트 자동 다운로드

장점: 사용자 셋업 부담 ↓, 일관된 결과
단점: 첫 실행 시 네트워크 필요, 디스크 ~10MB

## 4. 결론

이 샘플이 출판품질로 렌더링되면 스킬 도입 검증 성공입니다.

다음 항목이 깨지지 않아야 합니다:

1. **한글 자모 분리**: "ㅎ ㅏ ㄴ ㄱ ㅡ ㄹ" 같은 깨짐 없음
2. **줄바꿈**: 단어 중간에서 끊기지 않음 (`word-break: keep-all`)
3. **표 렌더링**: 셀 경계 깔끔
4. **코드 하이라이트**: 키워드 색상 적용
5. **페이지 번호**: 하단 중앙에 "1 / N" 표시
