# Workplace Conversations & Feedback

어려운 직장 대화와 건설적 피드백을 위한 구조화된 프레임워크.

## 핵심 프레임워크

### 1. Preparation-Delivery-Followup (PDF) 모델

| Phase | Focus | Key Questions |
|-------|-------|---------------|
| **Preparation** | 이슈 이해, 목표 정의, 감정 관리 | 문제가 뭔가? 원하는 결과? 침착한가? |
| **Delivery** | 중립적 시작, 사실 기반, 대화 유도 | 어떻게 시작? 증거? 참여 유도? |
| **Followup** | 합의 문서화, 체크인, 지원 | 뭘 합의했나? 언제 확인? |

### 2. SBI (Situation-Behavior-Impact) 피드백 모델

| Component | Description | Example |
|-----------|-------------|---------|
| **Situation** | 구체적 상황 | "어제 코드 리뷰에서..." |
| **Behavior** | 관찰 가능한 행동 | "...Sarah 설명 중 3번 끼어들었는데..." |
| **Impact** | 효과/영향 | "...아이디어 공유를 주저하게 되었습니다." |

---

## Preparation Phase

1. **이슈 명확화**
   - 구체적으로 무엇이 문제? (관찰 가능한 사실만)
   - 영향은? (나, 팀, 업무)
   - 패턴인가 일회성인가?

2. **감정 체크**

   | 감정 | 행동 전 |
   |------|---------|
   | 화남 | 24시간 대기, 쓰되 보내지 않기 |
   | 상처 | 중립적 사람과 먼저 대화 |
   | 불안 | 대화 연습 |
   | 방어적 | 나의 기여 파악 |

3. **목표 정의**
   - 원하는 결과?
   - 최소 수용 가능한 결과?
   - 양보할 수 있는 것?

## Delivery Phase

### Opening Lines

| Context | Opening |
|---------|---------|
| 일반 | "팀 성공에 중요한 이야기를 하고 싶습니다." |
| 성과 | "패턴을 발견해서 논의하고 싶습니다. 비판이 아니라 지원이 목적입니다." |
| 갈등 | "긴장이 있는 것 같아 당신 입장을 이해하고 싶습니다." |

### Facts, Not Blame

| Blaming | Factual |
|---------|---------|
| "프로젝트에 헌신적이지 않다" | "최근 3번 회의에서 업데이트가 짧았습니다. 워크로드에 문제가 있나요?" |
| "코드 품질에 신경 안 쓴다" | "이번 PR에서 QA가 12개 버그를 발견했습니다. 무슨 일인지 이야기해볼까요?" |

### 대화 유도

- "어떤 점이 어려웠나요?"
- "상황을 어떻게 보시나요?"
- "성공하려면 뭐가 도움이 될까요?"

## Followup Phase

- [ ] 합의된 액션 아이템 문서화
- [ ] 체크인 날짜 설정
- [ ] 지속적 지원 제공
- [ ] 개선 인정

---

## 대화 유형별 가이드

### 성과 피드백
- 구체적 예시로 시작
- 기대치/기준에 연결
- 미래 개선에 초점
- 지원과 리소스 제공

### 갈등 해결
- 먼저 각각 별도 미팅
- 공통 관심사 파악
- Win-win 해결책
- 합의 문서화

### 민감한 주제
- 비공개, 중립 장소
- 처리 시간 허용
- 직접적이되 동정적
- 기밀 존중

### 피드백 받기
1. 끝까지 듣기 (방어 준비 금지)
2. 명확화 질문
3. "제가 이해한 건..." 확인
4. 영향 인정
5. 감사 표현
