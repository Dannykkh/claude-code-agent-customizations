# Hermes 분석 프레임워크

## 영역 1: 비즈니스 모델

### Business Model Canvas (9 블록)

```
┌──────────────┬───────────────┬──────────────┬───────────────┬──────────────┐
│  핵심 파트너  │  핵심 활동     │              │  고객 관계     │  고객 세그먼트 │
│              │              │  가치 제안     │              │              │
│              │  핵심 자원     │              │  채널         │              │
├──────────────┴───────────────┼──────────────┴───────────────┴──────────────┤
│         비용 구조             │              수익원                         │
└──────────────────────────────┴────────────────────────────────────────────┘
```

각 블록을 3~5개 핵심 항목으로 채웁니다.

### Lean Canvas (스타트업용, 선택)

프로젝트가 초기 스타트업이면 Lean Canvas도 함께 작성:
- 문제 / 솔루션 / 핵심 지표 / 경쟁 우위 / 비용 / 수익

**출력:** 텍스트 캔버스 + Mermaid 다이어그램

## 영역 2: 수익 전략

### 가격 모델 분석

| 모델 | 적합 조건 | 장점 | 단점 |
|------|-----------|------|------|
| 구독형 (SaaS) | 반복 사용, B2B | 예측 가능 매출 | 이탈률 관리 필요 |
| 건당 과금 | 트랜잭션 기반 | 사용량 비례 | 매출 변동 큼 |
| 프리미엄 | 대중 시장 | 사용자 확보 용이 | 전환율 낮음 |
| 라이선스 | 기업용 | 높은 단가 | 영업 필요 |
| 광고 기반 | 대량 트래픽 | 무료 서비스 | 트래픽 의존 |

### 수익화 전략 브레인스토밍

3~5개 수익화 방안을 제시하고, 각각:
- 예상 수익 규모
- 구현 난이도
- 검증 실험 방법

| **SOM** | 1~3년 내 확보 가능 (Bottom-up) | {금액} |

## 영역 6: 코호트 분석 프레임워크

### 분석 설계

| 항목 | 설정 |
|------|------|
| **코호트 기준** | 가입 주 / 가입 월 / 첫 구매 월 |
| **측정 지표** | 리텐션율 / LTV / 전환율 |
| **분석 주기** | 주간 / 월간 |
| **세그먼트** | 유료/무료, 채널별, 지역별 |

### 코호트 테이블 템플릿

```
         Week 0  Week 1  Week 2  Week 3  Week 4
Cohort 1  100%    40%     25%     20%     18%
Cohort 2  100%    45%     30%     22%     --
Cohort 3  100%    42%     28%     --      --
```

### 단위 경제학 (Unit Economics)

| 지표 | 수식 | 목표 |
|------|------|------|
| **CAC** (고객 획득 비용) | 마케팅비 / 신규 고객 수 | {목표} |
| **LTV** (고객 생애 가치) | ARPU × 평균 이용 기간 | {목표} |
| **LTV/CAC** | LTV ÷ CAC | > 3x |
| **Payback Period** | CAC ÷ 월 ARPU | < 12개월 |
